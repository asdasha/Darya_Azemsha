    2. Синтаксис

    Естетвенно, что у HTML, как и у любого языка программирования, есть свой уникальный синтаксис, есть правила написания кода страниц, есть некоторые положения, которыми стоит руководствоваться при написании кода. Тем не менее, HTML прост в изучении, и, если следовать небольшому перечню правил, которые будут описаны ниже, все будет работать, как вами и было задумано: страница будет корректно отображаться в окнах браузеров.

    2.1. Документ HTML

    Документ HTML представляет собой простой текстовый документ в расширении *.html. Если открыть блокнот, и сохранить пустой документ в этом формате, то, когда вы откроете сохраненный файл, запустится не блокнот, а ваш браузер(т.е. программа для отображения веб-страниц), которым веб-страницы открываются по умолчанию. Естественно, что ничего ни в окне браузера, ни в заголовке окна не отобразится. С другой стороны, скопировав какой-то текст и вставив его в HTML-файл, результат окажется весьма и весьма непредсказуемым, потому что разные браузеры по-разному обрабатывают неправильный код (а, по сути, случайный текст, который вы вставили, и есть неправильный код с большей долей вероятности).

    Технология обработки браузером кода следующая: как только файл открыт в браузере - браузер считывает его содержимое и принимает решение, отображать считанную строку в рабочей области или нет. Изначально все содержимое файла по умолчанию считается HTML-кодом.

    Справедливо отметить, что разные браузеры по-разному отображают и правильно написанный код вследствие того, что они обладают разным функционалом и могут просто "не знать", как обрабатывать какой-то тег или его отдельную опцию, у них могут различаться некоторые значения по умолчанию и т.д. Это явление породило так называемую кросс-браузерную верстку.Хорошие новости в том, что сейчас наиболее распространенные браузеры поддерживают абсолютное большинство функций, задуманных в HTML и CSS. (Кстати, число функций постоянно растет: их разрабатывает W3C - Консорциум всемирной паутины)

    HTML-файл содержит код одной страницы, на которой расположены ссылки для перехода на другие страницы, которым соответствует другой документ. Однако бывают случаи, когда сайт состоит только из одной страницы. Например, применение различных Интернет-технологий может сделать так, что, хоть и сожержимое рабочей области постоянно меняется, весь код умещается в одном HTML-файле. Еще один вариант - создание интерактивных flash-приложений, являющихся, собственно, сайтами. Скорее всего, со временем появится что-то еще.

    2.2. Специфика кода HTML-страниц

    Что отличает код HTML от кода, написанного на других языках? Его теговая структура. Ее суть достаточно проста.

    Что такое тег? Тег - это ключевое слово, заключенное в треугольные скобки (например, <html>). Само ключевое слово определяет, как конкретно эта и следующие строки будут обрабатываться браузером. Еще существуют понятия открытия и закрытия тега. Открытый и закрытый тег выглядят примерно так - < тег ></ тег >, и пара открытого и закрытого тега образуют своеобразный блок кода.

    Допустим, внутри этого блока содержатся еще какие-то команды. В этом случае открытый тег начинает обработку кода внутри блока в соответствии с правилами, присущими тегу, а закрытый тег - для браузера признак того, что обработку по этим правилам надлежит прекратить. Из этого следует, что теги можно переносить на другую строку, или на несколько строк вниз, при этом код останется полностью функциональным.

    Также стоит отметить, что в пределах одной пары тегов можно расположить множество других пар. Для браузера это будет значить, что к последующим строкам нужно применить правила обработки всех открытых перед ним тегов. Pапись <html> body> <p> в переводе на человеческий будет означать что-то вроде "обрабатывать строки как HTML-код, в рабочей области браузера, в виде абзаца текста".

    Достоинство синтаксиса HTML в том, что можно решать сложные задачи разметки страницы, намного превосходящие возможности, допустим, текстовых процессоров, в которых нельзя создать таблицу в ячейке таблицы (... которая является ячейкой еще одной таблицы :). Недостаток его в том, что иногда возникают ошибки из-за невнимательности программиста к своевременному открытию и закрытию тегов, хотя по поведению браузера можно с легкостью узнать, в каком месте произошла ошибка, и как ее устранить.

    Следует отметить, что перекрывания тегов всегда лучше избегать. С одной стороны, код может правильно отобразиться в браузере. С другой стороны, запись <html> <head> <body> </head> </body> </html> (с любым кодом, содержащимся в тегах <head> и <body) - ошибочна, потому что тег <head> отвечает за содержимое заголовка окна браузера, а <body> - за рабочую область, и перекрывание этих тегов недопустимо. В слюбом случае, такой код не соответствует стандартам XHTML.

    Это и есть основные правила написания кода HTML. Тем не менее, в процессе программирования не стоит пренебрегать и читаемостью кода. Об этом читайте в главе 2.5.

    2.3. XHTML

    Собственно, XHTML - это строгий список правил, выполнение которых при написании кода гарантирует корректное отображение веб-страниц. XHTML - это не отдельный язык программирования, а стандарт.

    В соответствии с этим стандартом некоторые теги, например, считаются устаревшими и редко использующимися, в связи с чем код, их содержащий, проверку на соответствие XHTML не пройдет (процедура называется валидацией). Стандарты XHTML постоянно обновляются и к каждому из них есть подробное описание, ознакомиться с этой информацией может любой желающий. Последней версией XHTML является версия 2.0.

    2.4. Конструкция тега

    Как уже говорилось ранее, тег - это ключевое слово, заключенное в треугольные скобки. Теги открываются и закрываются по определенным правилам. Ключевое слово определяет, как обрабатываются строки, следующие за тегом. Однако список этих установленных по умолчанию правил можно изменять с помощью опций тега. В общем случае запись тега с одной или несколькими опциями выглядит так:

    < тег опция1 =" значение1 " опция2 =" значение2 " и т.д .> </ тег >

    Обратите внимание, что при закрытии тега не указываются никакие опции, а сами опции отделяются друг от друга и от ключевого слова тега одним (или несколькими) пробелами. В записи тега допускаются переносы строк, не разрывающее ключевые слова тегов и опций, но такие записи используются крайне редко.

    Также, есть теги, не содержащие в себе строк кода, например, тег переноса строки. Он записывается <br />, и в этом случае <br - открывает тег, а /> - его зыкрывает.

    2.5. Стиль программирования

    Простые HTML страницы содержат относительно небольшой код, но в некоторых случаях его объем превышает 1000 строк (и это далеко не предел), и при необходимости внести в него изменения возникают проблемы с поиском фрагмента кода, нуждающемся в редактировании. Следующие несколько советов помогут избегать подобных проблем и сделать ваш код более читаемым.

    Первое, на что хочется обратить внимание - это отступы. В процессе программирования получаются сложные структуры, соответственно, каждый открытый тег в пределах другого тега, стоит выделить табуляцией, что позволит вам ориентироватьс в уровнях вложенности тегов друг в труга. Например:

    <html>
	<head>
		<title>Страничка</title>
	</head>
	
	<body>
		<p>Добро пожаловать</p>
		<p>Это моя тестовая страничка, сделанная для примера,<br />
		иллюстрирующего то, как следует оформлять код</p>
	</body>
</html>

    Как видите, то, что теги <p> вложены в тег <body> и не вложены друг в друга, наглядно показано, и следует из самого кода. При этом код выполняется точно так же, как он был бы выполнен и без отступов: корректно и правильно.

    Этот же пример иллюстрирует еще несколько полезных советов по написанию кода. Крупные блоки кода следует отделять друг от друга одной или несколькими пустыми строками, чтобы было проще ориентироваться, где заканчивается один блок, и начинается другой. Отступы могут также наполняться комментариями, о которых будет сказано позднее.

    Теги, содержащие небольшой код, как, например, <title>Страничка</title>, можно записывать в строку, чтобы было использовано меньше строк и код выглядел компактнее. С другой стороны, теги, содержащие несколько строк кода, как, например, <body>, целесообразно открывать и закрывать на отдельных строках, потому что может возникнуть необходимость дописать к тегу несколько опций, и тогда код будет казаться громоздким.

    Придерживаясь этих простых правил может помочь вам лучше ориентироваться в том, что вы пишите, и любому человеку, которому нужно ваш код переписать.

    Кстати. Вообще, весь HTML код страницы можно записать в одну единственную строку, без переносов строк и отступов. Но, думаю, никому не взбредет в голову такая идея, потому что строка будет настолько длинной, что ориентироваться в написанном будет невероятно сложно. Но выглядит код - компактнее некуда :)

    3. Пошаговое создание веб-страницы

    В этой части учебника будет подробно описан процесс создания HTML-документа от самого начала - создания и сохранения документа, до самого конца - оформления и форматирования текста и таблиц. Будут применениы многие из возможностей HTML, будет проведен краткий обзор CSS-страниц, и по завершению работы над этой частью вы сможете создавать сложные веб-страницы.

    Не забывайте, что нужно не только понимать, что показано в примерах, но и следует самим пробовать создавать тестовые страницы, в которых содержится код, хотя бы чуть-чуть превосходящий по сложности примеры из книги. Поверьте, это очень ценный опыт.

    3.1. Создание документа

    Итак, начнем. Нам придется работать с HTML документом, описанию создания которого и посвящена эта глава. Для создания и работы с HTML-файлом совершенно не обязательно иметь какие-то специализированные среды программирования: вполне достаточно простого текстового редактора наподобие Блокнота. (Используя среды программирования, процесс написания кода будет несколько облегчен подсветкой ключевых слов, форматированием кода и автозаполнением тегов и опций).

    Открываем наш редактор. Видим пустой файл. Сохраняем его, в поле "имя файла" записываем [выбранное_имя].html. Готово! Теперь можете убедиться в том, что это веб-страница. Используя проводник или любой другой файловый менеджер откройте сохраненный вами файл. Так как веб-страницы по-умолчанию ассоциируются операционной системой с одним определенным веб-браузером, то в нем они и открываются.

    Итак, перед вами открытое окно браузера с двумя важными для нас элеметами: пустая рабочая область, и заголовок окна, в котором установлено имя файла, либо какое-то значение по умолчанию, потому что в нашем пустом документе не указано то, что должно быть напечатано в этой части окна.

    3.2. Начальная разметка страницы

    Перед тем, как начинать вносить информацию на нашу веб-страницу, необходимо правильно ее разметить для того, чтобы код, который мы будем записывать далее, корректно отображался. Откроем созданный нами HTML-файл и запишем туда следующие строки:

    <html>
	<head>
	</head>
	
	<body>
	</body>
</html>

    Итак, мы создали контейнеры, которые будут содержать: <html> - код, который браузер будет считать написанным на языке HTML, <body> - код, который будет обрабатываться и отображаться в рабочей области окна, и <head> - код, отвечающий за корректную загрузку страницы, подключение файлов, предоставление метаданных документа и т.д.

    Следует отметить, что в коде можно использовать больше одной пары тегов <html>, <body> и <head>. Фактически, содержимое этих пар будет обработано и отображено последовательно, как если бы код из одной пары скопировали и поместили после кода в другой паре. Однако, помните, что код, находящийся за пределами тега <html> браузером обрабатываться не будет.

    Чаще всего используется только по одному экземпляру тегов <html>, <head> и <body>. Если все-таки возникла такая необходимость, то целесообразным было бы оставить комментарий в какой-нибудь из строк кода. Комментарий - по сути, тоже тег, но он браузером не обрабатывается вообще, т.е. игнорируется. Синстаксис комментариев имеет вид:

    <!-- комментарий -->

    Комментарии могут быть размещены в любом месте кода без вреда для корректности отображения страницы. Допускаются переносы строк в комментариях, длина комментариев не ограничена, но не заывайте о том, что растет размер файла. Запишем в наш HTML-файл комментарий, в котором говортся, кто создал этот файл и занимался написанием страницы. Код будет иметь вид:

    <html>
	<!-- Эту страницу создал такой-то такой-то
	или такая-то такая-то -->
	<head>
	</head>
	
	<body>
	</body>
</html>

    Страница все еще пуста, но размечена, а сам код страницы сожержит информацию об авторе. Перейдем к заполнению тега <head>, отвечающего за заголовок окна.

    3.3. Наполнение тега <head>

    Тег <head> содержит строки кода, никак не отображающиеся в рабочей области браузера. Тем не менее, все они выполняют важные и четко-определенные функции, например, подключение дополнительных файлов к HTML-документу, определение метаданных документа, и т.д.

    Традиционно в тег <head> включают теги <base>, <link>, <meta>, <script>, <style> и <title>. Однако с течением времени многие из них потерияли важность, или их функции стали исполняться более эффективно с помощью других технологий. Сейчас часто используются два тега - <title> и <link>.

    Тег <title> отвечает за содержимое заголовка окна браузера. Пример его использования уже показывался в главе 2.4., продублируем его здесь:

    <html>
	<head>
                    <title>Страничка</title>
	</head>
	
	<body>
		<p>Добро пожаловать</p>
		<p>Это моя тестовая страничка, сделанная для примера<br />
		иллюстрирующего то, как следует оформлять код</p>
	</body>
</html>

    Результатом выполнения этого кода будет отображение в заголовочной части окна браузера надписи "Страничка". Следует отметить то, что содержимое тега <title> индексируется поисковыми машинами, то есть, при определенном содержимом заголовка и определенном поисковом запросе, страница имеет более высокие шансы быть показанной на странице результатов.

    Тег <link>, как вы могли подумать, не имеет ничего общего с гиперссылками. Он отвечает за подключение других файлов (чаще всего это .css файлы, отвечающие за оформление документа, и JavaScript-приложения, записанные в файл) и их обработку браузером параллельно обработке HTML-документа. Об использовании тега <link> будет подробно рассказано позже.

    Тег <base> ранее использовался для указания того, где расположены файлы, относительные ссылки на которые имеются HTML-коде. (Пример абсолютной ссылки - http://example.com/Folder1/Folder2/File.txt, относительной - /Folder1/Folder2/Fie.txt. Разница между ними в том, что абсолютная ссылка четко указывает расположение файла в файловой системе, а относительная - указывает расположение файла относительно каталога, в котором сейчас находится пользователь, либо файл с исполняемым кодом. Таким образом, если имеется картинка с относительной ссылкой /resources/image.jpg, а в теге <base> указан сайт "http://www.google.com" (синаксис такой записи - <base href = "http://www.google.com" />), браузер эту картинку будет искать по адресу http://www.google.com/resources/image.jpg)

    <meta> - тег, отвечающий за метаданные различного типа. Раньше в него записывались ключевые слова, которые обрабатывались поисковыми машинами, и, в зависимости от результата обработки, страница имела определенный рейтинг среди других результатов поиска. Порожденная этим проблема в рекламной области (некоторые программисты злоупотребляли использованием тега <meta> с целью цвеличения рейтинга страницы) вынудила исключить тег <meta> из списка индексируемых. Тем не менее, в теге <meta> можно указывать информацию об авторе документа, краткую аннотацию к странице, или ее описание. Эта информация может оказаться кому-то полезной.

    Тег <script>, как вы уже знаете, отвечает за исполняемые на странице сценарии. Однако сейчас все чаще тег <script> выносится за рамки тега <head>, так как он может быть размещен вообще в любой части документа.

    Тег <style> отвечает за стиль оформления конкретно того документа HTML, в котором содержится этот тег. Его используют все реже иреже в связи с тем, что появились так называемые каскадные таблицы стилей. Стили, применяющиеся на многих страницах записываются в .css файл, который подключается в случае необходимости к документу с помощью тега <link>.

    Вернемся к создаваемой нами странице, все еще пустой, но размеченной. Запишем в нее строку кода, содержащую тег <title>. Код будет выглядеть вот так:

    <html>
	<!-- Эту страницу создал такой-то такой-то
	или такая-то такая-то -->
	<head>
		<title>
			Заголовок страницы, который вы придумаете сами :)
		</title>
	</head>
	
	<body>
	</body>
</html>

    Итак, у нас есть размеченная страница, и уже имеющая заголовок. Перейдем к наполнению нашей страницы текстом.

    3.4. Добавление и форматирование текста

    Текст, который должен отобразиться в окне браузера, записывается в теге <body>.

    Вообще, для того, чтобы текст был отображен, не нужно использование каких-то специальных тегов. Корректно отобразится вот такой код:

    <html>
	<!-- Страница, создаваемая по учебнику -->
	<head>
		<title>
			Тестовая страница
		</title>
	</head>
	
	<body>
                    Маленький текстик или полный текст романа "Война и мир"
	</body>
</html>

    Текс будет со стандартным форматированием (чаще всего это гарнитура Times New Roman, без отступов, показателей веса и с размерм кегля 12-14 пунктов в зависимости от используемого браузера, цвет текста - черный), и будет расположен в левой верхней части рабочей области.

    Есть различные способы разметки и верстки текста. Специальное форматирование с изменением гарнитуры шрифта, цвета символов и других характеристик будет рассмотрено позже: здесь показаны только базовые стили, одинково отображающиеся всеми браузерами. Эти базовые стили получаются с использованиемследующих тегов: <p>, <h1>, <h2> ... <h6>, <b>, <em>.

    Тег <p> - (от англ. paragraph - абзац) формирует абзац текста. По сути, этот тег формирует лишь блок текста, а после этого блока делается небольшой отступ (величина которого может быть установлена). Теги <h1> - <h6> - (от англ. heading - заголовок) формирует текст, оформленный как заголовок (шесть тегов - шесть вариантов заголовков). <b> делает текст жирным, <em> - наклонным. Следующий пример иллюстрирует применение этих тегов.

    <html>
	<body>
		<p>Абзац текста</p>
		<p>Еще абзац</p>
		<h1>Заголовок 1</h1>
		<h2>Заголовок 2</h2>
		<h3>Заголовок 3</h3>
		<h4>Заголовок 4</h4>
		<h5>Заголовок 5</h5>
		<h6>Заголовок 6</h6>
		<b>Жирный текст</b>
		<em>Наклонный текст</em>
		<em><b>Жирный и наклонный,</b>а здесь только наклонный</em>
	</body>
</html>
    3.5. Таблицы в HTML

    Одним из основных способов отображения данных и верстки страниц является использование таблиц. Мы привыкли видеть таблицы как способ работы с числовыми данными, но в контексте верстки, например, рабочую область окна браузера можно рассматривать как большую ячейку таблицы 1х1.

    Построение и использование таблиц очень просто, для этого нужно использовать теги <table>, <tr> и <td>. Первый из представленных - <table> -  инициализирует таблицу. Второй - <tr> - стоздает в таблице одну строку. Третий - <td> - создает одну ячейку в созданной строке.

    На первый взгляд все это может показаться сложным. Темне менее, рассмотрим пример создания таблицы 3х3. 

    <table>
	<tr>
		<td></td>
		<td></td>
		<td></td>
	</tr>
	<tr>
		<td></td>
		<td></td>
		<td></td>
	</tr>
	<tr>
		<td></td>
		<td></td>
		<td></td>
	</tr>
</table>

    Итак, мы видим: тег <table>, создающий таблицу, в нем три созданные строки, и в каждой строке по три ячейки. Таким образом мы создали таблицу 3х3.

    Вы заметили, что мы не создавали столбцов таблицы. Явно – нет. Но в каждой строке мы указали по три ячейки, а ячейки, составленные одна под другой , образуют столбец. Да, это несколько неудобно, но, тем не менее, является тем, как все устроено в HTML.

    Интересный момент. Есть несколько способов задать всем ячейчас столбца какое-то свойство. Первый – в каждом теге <td>, соответствующем интересующей нас ячейке, дописывать какое-то свойство. Двугой способ – запись тегов <col /> или <colgroup>

    Также иногда мы сталкиваемся с необходимостью объединить ячейки в таблице. Тогда мы в интересующих нас строках или столбцах указываем атрибуты colspan или rowspan:

    <table>
    <tr>
    <td colspan=2></td>
    </tr>
    <tr>
    <td></td><td></td>
    </tr>
    <tr>
    <td></td><td></td>
    </tr>
    <tr>
    <td></td><td></td>
    </tr>
    </table>

     

    Как видим, у нас есть таблица из четырех строк и двух столбцов, и, как вы могли догадаться, мы объединили две ячейки первой строки. Обратите внимание, что теперь мы в HTML-коде пишем не две-  а только один тег <td>. Переводя на человеческий язык мы получим что-то вроде: «В первую строку таблицы положить одну ячейку, которая объединяет (ага, colspan) две ячейки».

    При объединеии ячеек в столбце наблюдаем немного иную картину.

    <table>
    <tr>
    <td rowspan=3></td>
    <td></td>
    <td></td>
    </tr>
    <tr>
    <td></td>
    <td></td>
    </tr>
    <tr>
    <td></td>
    <td></td>
    </tr>
    </table>

    Браузер при обработке этого кода считает строку содержащую rowspan=3 и создаст ячейку размером 3х1, затем в первую строку добавит еще два столбца, и перейдет к заполнению следующей строки. Естественно, что в начале у нас стоит наша ячейка 3х1, соответственно продолжит браузер создавать таблицу со следующей позиции.

    Вот и все, что касается создания таблиц. Полдробнее о тегах, и об их дополнительных возможностях вы можете прочитать в глоссарии.


    Нередко программисту приходится создавать списки. Наихудший вариант в этом случае – это использование, например, символов «минус» (-) или других символов для обозначение очередного элемента списка. Наихудший он потому, что HTML предусмотрены теги для создания «правильных» списков.

    Начнем с того, что существуют три типа списков: нумерованные, ненумерованные и списки определений. Остановимся чуть подробнее на каждом и них.

    Нумерованные списки нам давно знакомы – это набор каких-то объектов, каждому из которых присвоен порядковый номер, и в соответствии с этими номерами они поставлены друг относительно друга. Для определения нумерованного списка используем тег <ol> - “ordered list”. А для каждого из элементов этого списка будем использовать тег <li> - “list item”.

    <ol>
      <li>Coffee</li>
      <li>Tea</li>
      <li>Milk</li>
    </ol>

    Получим список, в котором будут перечислены Кофе, Чай и Молоко.

    Ненумерованные списки отличаются от нумерованных тем, что вместо порядкового номера либо другого счетчика выступает т.н. «буллит» - по сути любой символ, который отобразится перед каждым из элементов списка. Код отличается от предыдущего одной буквой: вместо тега <ol> используем <ul> - “unordered list”.

    <ul>
      <li>Coffee</li>
      <li>Tea</li>
      <li>Milk</li>
    </ul>

    Списки определений, очевидно, это перечисленные термины и их расшифровки. Создание таких списков сильно отличается от двух других типов. Теги, используемые в этом случае - <dl> для инициализации списка, <dt> для указания термина, <dd> для его расшифровки. И да, у одного термина может быть несколько определений

    <dl>
    <dt>Coffee</dt>
    <dd>- black hot drink</dd>
    <dd>- color</dd>
    <dd>- white cold drink</dd>
    <dt>Milk</dt>
    <dd>- white cold drink</dd>
    </dl>

    Опять же, возможности списков расширяются за счет использования различных опций,полный список которых вы можете найти в глоссарии, поэтому заглядывайте туда.

    Если на какой-то из страниц вашего сайта пользователь на нее пришедший должен будет вводить данные, вы, скорее всего, прибегнете к испольхованию форм. Формы – это еще один объект, который может включать в себя множество элементов управления, такие как текстовые поля, кнопки, чекбоксы, и прочие.

    Собственно, создание формы очень просто и выглядит следующим образом.

    <form></form>

    Формам чаще всего присваивается имя с помощью опции id, и указывается место ее обработки: на клиентской машине или на сервере, но все это имеет мало общего с HTML  и версткой. Пока вам достаточно знать, что форма – просто объединяет в себя элементы интерфейса.

    Для создания кнопки, выпадающего меню и текстового поля используется один и тот же тег - <input>,  потому что все они, по сути, являются элементами ввода. Но, так как все элементы ввода имеют разный тип, то он указывается опцией в этом теге.

    <input type="text" name="fname" />
    <input type="submit" value="Submit" />

    Этот код генерирует два элемента ввода – текстовое поле и кнопку. Есть много других объектов для ввода данных, и будет лучше, если вы потренируетесь сами их создавать. Возможные значение для опции type перечислены в глоссарии.


     

    Гиперссылки – удобный способ связи одного документа с другим, а так же для навигации в пределах одного документа. Для создания гиперссылки используется тег <a>. Важная опция, которая указывает путь, по которому следует перейти, записывается внутри тега как href='' '', где внутри кавычек записана либо абсолютная, либо относительная (об этом было рассказано выше).

    <a href=”http://www.google.com”>google</a>

    Обработанный данный код будет отображен слудеющим образом. Появится слово google, оформленное как ссылка, и по клику на нем откроется страница поисковой системы google. Заметно, что между открытием и закрытием тега <a> следует поместить текст (либо другой объект), содержащий ссылку.

    Третий вариант использования ссылок – это перемещение внутри документа. Эта возможность связана также с опцией id, присущей почти всем тегам. Предположим, у нас есть абзац текста, которому мы присвоили id=”test_par”. Также есть ссылка, записанная следующим образом:

    <a href=”#test_par”>К абзацу</a>

    Когда мы перейдем по ссылке, страница в браузере прокрутится автоматически до того места, где находится искомый абзац.

    Определение и использование

    Определяет комментарий. Тег "комментарий" используется для вставки комментариев в исходный код. Комментарий будет игнорироваться браузером. Вы сможете использовать комментарий, чтобы объяснить свой код. Комментарий поможет вам изменить исходный код через некоторое время, если вы забудете все его детали. Вы можете также хранить код программы или конкретной информации внутри комментария. В этом случае информация внутри комментария не будет отображаться для пользователя, но все еще будет доступна в программе.

    Тег "комментарий" не поддерживает ни один из стандартных атрибутов.

    Тег "комментарий" не поддерживает ни один из атрибутов события.

    Тег <!DOCTYPE>

    Определение и использование

    Определяет тип документа.

    Декларация DOCTYPE должна быть самым первым, что высвечивается в документе HTML, до <html> тега. Эта конструкция не является тегом HTML, это инструкция к веб-браузеру о том, что версия языка разметки страницы совпадает. Она ссылается на Определение Типа Документа (ОТД). ОТД задает правила для языка разметки, так чтобы браузеры могли показать содержание страницы правильно.

    Тег <a>

    Определение и использование:

    Тег <a> определяет якорь (фиксатор). Якорь можно использовать с двумя целями: чтобы создать ссылку на другой документ, с помощью атрибута href, или чтобы создать закладку в документе, используя имя атрибута. Этот элемент обычно упоминают как ссылка или гиперссылка. Наиболее важным атрибутом <a> является элемент href атрибута, который указывает назначение ссылки. По умолчанию ссылки будет выглядеть следующим образом во всех браузерах:

    • Непосещенные ссылки - подчеркнуты и синего цвета;
    • Посещенные ссылки - подчеркнуты и фиолетового цвета;
    • Активная ссылка - указывается и красного цвета.

    Опциональные атрибуты:

    Атрибут Значения Описание
    charset char_encoding Определяет набор символов из привязанного документа
    coords coordinates Определяет координаты ссылки
    href URL Определяет направление ссылки
    hreflang language_code Указывает язык привязанного документа
    name section_name Определяет имя якоря (фиксатора)
    rel text Определяет связь между текущим документом и привязанным
    rev text Определяет отношения между привязанным документом и текущим
    shape default
    rect

    target _blank
    _parent

    framename     		Указывает, где открыть привязанный документ

    Тег <object>
    

    Определение и использование:
    

    Тег <object> определяет внедренный объект. Тег <object> используется для включения объектов, таких как изображения, аудио, видео, Java-апплеты, ActiveX, PDF и Flash.
     Объект поддержки в браузерах зависит от типа объекта. К сожалению, основные браузеры используют различные коды для загрузки одного типа объекта.
     К счастью, элемент объект обеспечивает решение. Если элемент объект не отображается, код между <object> и </ object> тегами будет выполнен.
     Таким образом, мы может иметь несколько вложенных элементов объекта (по одному для каждого браузера).

    onclick script Скрипт выполняется по щелчку мыши
    ondblclick script Скрипт выполняется по двойному щелчку мыши
    onmousedown script Скрипт запускается после нажатия кнопки мыши
    onmousemove script Скрипт выполняется при движении курсора
    onmouseout script Скрипт выполняется, когда курсор находится вне элемента
    onmouseover script Скрипт выполняется при движении курсора над элементом
    onmouseup script Скрипт выполняется при отпускании кнопки мыши
    onkeydown script Скрипт выполняется при нажатии клавиши
    onkeypress script Скрипт выполняется после нажатия клавиши
    onkeyup script Скрипт выполняется при отпускании клавиши

    Тег <var>

    Определение и использование:

    Тег <var> определяет переменную часть текста.

    Стандартные атрибуты:

    Атрибут Значения Описание
    class classname Определяет имя класса для элемента
    dir rtl
    ltr     		Определяет направление текста для содержания в элемент
    id id Определяет уникальный идентификатор для элемента
    lang language_code Указывает код языка за содержание в элемент
    style style_definition Задает встроенный стиль для элемента
    title text Указывает дополнительную информацию об элементе
    xml:lang language_code Указывает код языка за содержание в элемент, в XHTML документах
